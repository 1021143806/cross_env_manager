## 二、分层说明

### 2.1 路由层 (routes/)

| 蓝图文件 | 职责 | 路由数 |
|---------|------|--------|
| `auth_routes.py` | 登录/注销/状态 | ~5 |
| `template_routes.py` | 模板 CRUD、搜索、子任务管理、设备同步 | ~25 |
| `task_routes.py` | 任务查询、重发、异常完成、操作日志 | ~15 |
| `config_routes.py` | 调度配置备份/恢复 | ~15 |
| `system_routes.py` | 系统配置、健康检查、数据库切换 | ~20 |
| `dispatch_routes.py` | 调车看板、自动调度、自愈 | ~30 |
| `stats_routes.py` | 统计数据 API | ~10 |
| `docs_routes.py` | 文档页面渲染 | ~5 |
| `monitor_routes.py` | 系统监控页面 | ~5 |
| `custom_table_routes.py` | 定制表编辑 | ~15 |
| `join_qr_routes.py` | 交接点配置 | ~8 |
| `platform_switch_routes.py` | 平台切换 | ~5 |
| `system_upgrade_routes.py` | 升级管理 | ~4 |

**注册方式**: `routes/__init__.py` → `register_blueprints(app)` → 在 `app.py` 启动时调用

### 2.2 业务逻辑层 (services/)

| Service | 说明 |
|---------|------|
| `AuthService` | 用户认证、管理员提权、RCS 账号验证 |
| `TemplateService` | 模板 CRUD、搜索、复制、子任务管理 |
| `ConfigService` | 调度配置读写、备份管理 |
| `SystemConfigService` | 系统配置（env.toml）读写、备份管理 |
| `StatsService` | 统计查询、缓存 |
| `DispatchService` | 调车调度逻辑、自愈 |
| `DeviceSyncService` | 设备同步（SSE 流式） |
| `JoinQRService` | 交接点配置管理 |
| `PlatformSwitchService` | 平台切换 |
| `UpgradeService` | 升级管理（备份、解压、回滚） |

### 2.3 数据访问层 (dao/)

| DAO | 说明 |
|-----|------|
| `BaseDAO` | 统一连接池管理（DBUtils）、增删改查封装 |
| `TemplateDAO` | 模板相关 SQL |
| `DetailDAO` | 子任务明细相关 SQL |

### 2.4 中间件 (middleware/)

| 模块 | 说明 |
|------|------|
| `cache.py` | Flask-Caching 缓存，TTL=5min，写操作自动清除 |
| `exception.py` | 统一异常：`AppError`、`NotFoundError`、`AuthError`、`ValidationError` |

### 2.5 工具模块 (modules/)

| 模块 | 说明 |
|------|------|
| `database/connection.py` | 连接池管理（DBUtils PooledDB） |
| `custom_table/` | 配置驱动的跨库表编辑 |
| `query/` | 任务查询扩展逻辑 |

## 四、关键数据流

### 4.1 任务下发流程

```
用户选择区域+任务 → 前端校验容量 → POST ICS API
     → ICS 返回 orderId → 记录到 localStorage
     → 15 秒轮询查询状态 → 状态 8 自动清理
```

### 4.2 跨环境任务调度

```
调车看板定时触发 → 检查区域平衡 → 检查空车限制
     → 调用 ICS API 下发 → 记录到本地 JSON
     → 自愈线程检查超时 → 超时取消 + 重发
```

### 4.3 升级流程

```
上传 ZIP → 备份项目 → 解压覆盖 → 记录版本 → supervisorctl 重启
```

## 五、启动流程

```python
app.py 启动:
  1. 解析命令行参数 (config 路径, port)
  2. 初始化 Flask app + Jinja2 + session
  3. 注入上下文 (app_version, config_path)
  4. register_blueprints(app)  ← 注册所有路由
  5. 初始化连接池
  6. 注册异常处理器
  7. 启动调度线程 (自愈 + 轮询)
  8. app.run(port=5000)
```
