# 华睿相关开发要求

> 最后更新: 2026-05-18

## 1. ICS 任务状态码

| status | 含义 | 说明 |
|--------|------|------|
| 3 | 已取消 | 任务被取消 |
| 6 | 开始执行 | 设备已接任务 |
| 7 | 设备忙 | RCS 返回，设备非空闲 |
| 8 | 已完成 | 任务正常完成 |
| 9 | 执行中 | ICS 中间状态 |
| 10 | 执行中（未知） | ICS 可能只上报 10 不上报 8 |

**调车模块处理**：
- `status in (6, 9, 10)` → 视为执行中，加入模板 JSON
- `status == 8` → 视为完成，清理模板 JSON，触发 `_update_daily_stats` 统计
- `status == 3` → 已取消，清理模板 JSON，currentCount +1

## 2. ICS 设备状态

| ICS state | 前端映射 | 含义 |
|-----------|---------|------|
| Idle | idle | 空闲 |
| InTask | busy | 执行中 |
| Charging | charging | 充电中 |
| Offline | Offline | 离线 |
| Downlined | Downlined | 下线 |

**空车下发规则**：只能选 `state == 'Idle'` 的设备。InTask/Charging/Offline/Downlined 的设备不应被下发空车任务。

## 3. ICS 上报配置要求

### 3.1 上报接口
```
POST /api/dispatch/report_status
```

### 3.2 上报时机
- **status=6**：设备接任务时上报
- **status=8**：任务完成时上报（**必须配置**，否则折线图统计为 0）
- **status=9/10**：中间状态上报（可选）

### 3.3 上报报文格式
```json
{
    "modelProcessCode": "模板code",
    "deviceCode": "设备序列号",
    "deviceNum": "设备编号",
    "status": 8,
    "orderId": "订单号",
    "subTaskId": "子任务ID",
    "shelfNumber": "货架号"
}
```

## 4. 调车模块关键逻辑

### 4.1 空车下发流程
1. `calculate_area_balance` 计算各区域 need
2. `_execute_dispatch` 执行下发
3. 空车回（out）：`_select_device_for_empty_return` 选设备
   - 排除 pending 中的设备
   - 排序：最近未下发优先 → 电量低优先 → idle 优先
   - 逐个实时查询 ICS 确认 Idle
   - 全部非 Idle → 不指定设备
4. 空车来（in）：不指定设备

### 4.2 状态上报处理
1. 按 template code 匹配区域和模板
2. 匹配成功 → 更新/清理模板 JSON
3. 匹配失败 → 按 order_id 跨区域匹配
4. status=8 完成 → 调用 `_update_daily_stats` 统计

### 4.3 自恢复检查
- 间隔：默认 300 秒
- 全量查询 ICS 设备 API，更新 `currentCount.json` 中的 state 和 battery
- 清理离线/下线设备
- 自动添加新上线设备

### 4.4 解死锁
- 只取消 `deviceCode` 为空的空车任务（未分配到设备的）
- 已分配到设备的任务不取消

## 5. 常见问题

### Q: 折线图统计为 0
- 检查 ICS 是否配置了 status=8 完成上报
- 检查模板 code 是否在调车模块配置中存在
- 如果模板不在配置中，按 order_id 匹配清理也会统计（v2.1.11+）

### Q: 设备网格状态不更新
- `currentCount.json` 的 state 由自恢复检查更新（默认 5 分钟间隔）
- 空车下发前会实时查询确认 Idle
- 如果设备一直显示"查询失败"，检查 `device_query_api` 配置

### Q: 空车任务下发到 InTask 设备
- 下发前会实时查询 ICS 确认 Idle
- 非 Idle 设备会被跳过
- 如果仍有问题，检查 `device_query_api` 的 areaId 是否正确

## 6. 配置注意事项

### device_query_api
- 格式：`http://{server_ip}:7000/ics/out/device/list/deviceInfo`
- 含 `XX` 占位符视为未配置
- areaId 必须与 ICS 中配置一致

### 模板 code
- 必须与 ICS 中 `modelProcessCode` 完全一致
- 未配置的模板走 order_id 匹配（功能降级）


# 华睿提供的接口

## 设备预占接口

单台设备可以被预占多次，多次重复预占返回应对皆为成功，离线设备、已下线设备都可以提前预占，不影响，但设备不存在时预占，即查询不到设备序列号时才会报错。

设备重启以及重新注册平台不会影响预占状态。



> post 请求地址：ip:7000/enableAGV

```JSON
{
  "areaId":1,//子区域id
  "enable": 1,
  "robotId": "CC06659BAK00004",//设备序列号
  "lastUpdateTime": 1776761753960,//最后一次更新时间
  "stay": 1
}
```


> 响应

```JSON
{
    "code": 1000,
    "desc": ""
}
```
