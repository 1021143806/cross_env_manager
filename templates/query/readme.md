# 跨环境任务查询 · 使用帮助

## 1. 任务单号查询

输入完整的 OrderId（例如 `pad_html2026-04-20 14:30:00`），系统会：

- 从目标服务器获取主任务信息
- 从本地数据库查询 `task_group` 和 `task_group_detail` 完整记录
- 展示关联的大模板及子模板，可点击跳转编辑
- 列出该模板下其他执行中/异常任务
- 聚合所有子任务信息（跨服务器），并以 Markdown 风格展示
- **自动查询设备实时状态**：从子任务中提取设备序列号，查询各服务器上的设备状态并展示设备状态卡片

## 2. 设备号查询

输入设备编号（如 `DJC2`），系统会：

- 按状态优先级（执行中→失败→重发中→已下发→发送中→已取消→未发送→已完成）查询该设备最近的 **5条任务**
- 默认展示第一条任务的完整深度查询结果
- 顶部显示**最近任务标签栏**，点击可快速切换查看其他任务
- 展示设备实时状态卡片（各服务器上的设备状态、电量、区域等）

## 3. 货架号查询

输入货架编号，系统按优先级（执行中 → 失败/异常 → 已完成）查找该货架最近的任务，然后自动转为任务单号查询，同样展示设备实时状态。

## 4. IP地址输入

支持完整IP如 `10.68.2.32` 或仅最后一段数字 `32`，系统会自动补全为 `10.68.2.32`。

## 5. 查看原始JSON

点击各卡片上的 <i class="bi bi-info-circle"></i> 按钮，可查看对应接口返回的原始数据。

## 6. 任务重发

### 重发按钮
子任务卡片底部根据状态显示不同操作按钮：

| 子任务状态 | 按钮 | 说明 |
|-----------|------|------|
| 非6（执行中）且非8（已完成） | 🔄 重发 | 先预占设备→取消任务→再预占→修改数据库状态 |
| 5（重发中） | ✅ 异常完成 | 仅将子任务状态改为3（已取消），用于处理卡住的重发任务 |

### 重发流程（8步）
1. **查询子任务信息**：获取 service_url、device_code、area_id
2. **前置任务检查**：检查上一个子任务是否还在执行中，如果是则阻止重发
3. **检查大模板状态**：状态6（执行中）和8（已完成）不允许重发
4. **检查子模板状态**：状态6（执行中）和8（已完成）不允许重发
5. **预占设备（第一次）**：调用 `POST /enableAGV` 预占对应服务器的设备
6. **取消任务**：调用 `POST /ics/out/task/cancelTask` 取消远端任务
7. **预占设备（第二次）**：再次调用 `POST /enableAGV` 确保预占状态
8. **生成新 sub_order_id**：子ID+1
9. **修改数据库状态**：大模板状态→5，子模板状态→5

### 渐进刷新
重发成功后，前端自动渐进刷新：
- 2秒后刷新 → 如果状态还是5（重发中）→
- 再等3秒后刷新 → 如果还是5 →
- 再等4秒后刷新
- 状态变为非5（如6执行中/8已完成）则停止刷新

### 注意事项
- 重发操作直接修改生产数据库，请谨慎使用
- 如果前置任务仍在执行中，需要先去对应服务器处理
- 设备预占失败会阻断重发，提示"设备未添加至当前服务器"
- 频繁点击刷新按钮会触发防抖警告（1秒内只能刷新一次）

## 7. 今日任务状态概览

左侧面板底部展示当天大模板任务的状态分布：

- **统计面板**：展示所有状态（容量管控、已被异常完成、正在发送、重发中、已下发、任务失败、任务完成）的数量和百分比
- **饼图**：展示异常状态（status=3 已被异常完成、status=7 任务失败）按错误描述的分类分布
- 点击刷新按钮可手动更新数据

## 8. 高级选项

- **查询本地数据库**：同时查询本地 `task_group` 和 `task_group_detail` 表
- **获取模板关联信息**：获取跨环境模板的详细信息
- **查询同模板其他执行中任务**：列出同一模板下其他正在执行或异常的任务
