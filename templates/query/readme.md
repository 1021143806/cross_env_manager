# 跨环境任务查询 · 使用帮助

## 1. 任务单号查询

输入完整的 OrderId（例如 `pad_html2026-04-20 14:30:00`），系统会：

- 从目标服务器获取主任务信息
- 从本地数据库查询 `task_group` 和 `task_group_detail` 完整记录
- 展示关联的大模板及子模板，可点击跳转编辑
- 列出该模板下其他执行中/异常任务
- 聚合所有子任务信息（跨服务器），并以 Markdown 风格展示
- **自动查询设备实时状态**：从子任务中提取设备序列号，查询各服务器上的设备状态并展示设备状态卡片

## 2. 设备号查询

输入设备编号（如 `DJC2`），系统会：

- 按状态优先级（执行中→失败→重发中→已下发→发送中→已取消→未发送→已完成）查询该设备最近的 **5条任务**
- 默认展示第一条任务的完整深度查询结果
- 顶部显示**最近任务标签栏**，点击可快速切换查看其他任务
- 展示设备实时状态卡片（各服务器上的设备状态、电量、区域等）

## 3. 货架号查询

输入货架编号，系统按优先级（执行中 → 失败/异常 → 已完成）查找该货架最近的任务，然后自动转为任务单号查询，同样展示设备实时状态。

## 4. IP地址输入

支持完整IP如 `10.68.2.32` 或仅最后一段数字 `32`，系统会自动补全为 `10.68.2.32`。

## 5. 查看原始JSON

点击各卡片上的 <i class="bi bi-info-circle"></i> 按钮，可查看对应接口返回的原始数据。

## 6. 任务重发

### 重发按钮
子任务卡片底部根据状态显示不同操作按钮：

| 子任务状态 | 按钮 | 说明 |
|-----------|------|------|
| 非6（执行中）且非8（已完成） | 🔄 重发 | 先预占设备→取消任务→再预占→修改数据库状态 |
| 5（重发中） | ✅ 异常完成 | 仅将子任务状态改为3（已取消），用于处理卡住的重发任务 |

### 重发流程（9步，SSE 实时动画）

点击重发按钮后，弹出进度模态框，通过 SSE 实时推送每步执行结果：

```
步骤0: 查询子任务信息
  ├─ 从 fy_cross_task_detail 查询子任务记录
  ├─ 如果子任务不存在，从 fy_cross_task 主任务补充 device_code
  ├─ 从 fy_cross_model_process_detail.template_code 提取 area_id（最后一段数字）
  └─ 从同 order_id 的其他子任务获取 service_url

步骤1: 前置任务检查
  └─ 检查 task_seq-1 是否还在执行中（状态4/6/9），是则阻断

步骤2: 检查大模板状态
  └─ 状态6（执行中）和8（已完成）不允许重发

步骤3: 检查子模板状态
  └─ 子任务存在时检查状态6和8，不存在则跳过

步骤4: 预占设备（第一次）
  ├─ POST http://{server_ip}:7000/enableAGV
  ├─ 参数: {areaId, enable:1, robotId, lastUpdateTime, stay:1}
  └─ 失败则阻断，提示"设备未添加至当前服务器"

步骤5: 取消任务
  ├─ POST http://{server_ip}:7000/ics/out/task/cancelTask
  ├─ 参数: [{orderId: sub_order_id, destPosition: ""}]
  └─ code=8007（订单不存在）也视为成功

步骤6: 预占设备（第二次）
  └─ 重复步骤4，确保预占状态

步骤7: 生成新 sub_order_id
  └─ 子ID+1（如 _1067 → _1068）

步骤8: 修改数据库状态
  ├─ UPDATE fy_cross_task SET task_status=5 WHERE orderId='...'
  └─ UPDATE fy_cross_task_detail SET status=5 WHERE sub_order_id='...'
```

每步显示：
- ✅ 完成 + 耗时(ms) + 请求/响应报文
- ❌ 失败 + 错误信息
- 进度条 + 步骤计数（x/9）
- 全部完成后不关闭模态框，1s 后自动刷新页面

### 注意事项
- 重发操作直接修改生产数据库，请谨慎使用
- 如果前置任务仍在执行中，需要先去对应服务器处理
- 设备预占失败会阻断重发，提示"设备未添加至当前服务器"
- 取消任务返回"订单不存在"（code=8007）视为成功
- 子任务记录不存在时，跳过子模板状态检查，仅更新大模板状态

## 7. 今日任务状态概览

左侧面板底部展示当天大模板任务的状态分布：

- **统计面板**：展示所有状态（容量管控、已被异常完成、正在发送、重发中、已下发、任务失败、任务完成）的数量和百分比
- **饼图**：展示异常状态（status=3 已被异常完成、status=7 任务失败）按错误描述的分类分布
- 点击刷新按钮可手动更新数据

## 8. 高级选项

- **查询本地数据库**：同时查询本地 `task_group` 和 `task_group_detail` 表
- **获取模板关联信息**：获取跨环境模板的详细信息
- **查询同模板其他执行中任务**：列出同一模板下其他正在执行或异常的任务
